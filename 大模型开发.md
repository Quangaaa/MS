一、整体定位（RAG 标准链路）

plaintext




用户问题 Query
👉 初检索（粗召回/粗排）：从海量知识库快速筛一批候选
👉 重排序 Rerank（精排）：对少量候选高精度打分筛选

核心逻辑：初检索保速度、兜底召回；重排序保精度、降噪提纯
二、初检索 / 粗检索（Coarse Retrieval）

1. 核心作用

面对几万 / 百万级海量文档，无法全量精准计算；
用轻量化、低消耗算法，快速从全库捞出一批相关候选片段；
优先保证召回率（别漏掉有用内容），适当牺牲精准度；
输出：固定 Top-K（业内常用 Top-30 ~ Top-100）交给重排。
2. 常见主流方法

（1）词法检索（稀疏检索）

只匹配字面、关键词、实体
BM25【工业最常用】：TF-IDF 升级版，考虑文档长度、词权重，文档检索标配
TF-IDF：传统词频匹配，效果弱，老旧项目在用
✅ 优势：专有名词、专业术语、编号、关键词命中极强❌ 劣势：完全不懂语义，同义 / 改写问题匹配不到
（2）语义向量检索（稠密检索）

流程：Query & 文档分别通过 Embedding 模型生成向量 → 向量相似度匹配
模型：BGE、M3E、Sentence-BERT
向量库索引：
暴力余弦（小库）
ANN 近似检索（大库必用）：HNSW、IVF、PQ 量化（Milvus/ES/FAISS 内置）
✅ 优势：理解语义、同义、上下文泛化❌ 劣势：关键词 / 实体容易丢失，存在语义漂移
（3）混合检索（Hybrid 召回）【企业标配】

BM25 + 向量检索 结果融合互补优缺点，同时保证关键词 + 语义，RAG 基础效果拉满
3. 耗时范围（线上实测参考）

BM25 词法检索：1～5 ms
Embedding 生成 + 向量 ANN 检索：10～50 ms
混合检索（BM25 + 向量）：20～60 ms
4. 核心特点

计算轻、并发高、吞吐大
近似匹配、存在噪声、相关度打分粗糙
必须控制召回数量，不能直接喂给大模型
三、重排序 / Rerank（Fine Ranking 精排）

1. 核心作用

只接收初检索筛选后的几十条小批量候选，不碰全量数据；
用更强的语义模型，对 Query + 文档做交叉深度语义匹配；
精准打分、过滤垃圾片段、剔除弱相关、乱序内容；
输出：筛选出高质量 Top-N（常用 Top-3 ~ Top-10），压缩上下文，减少 LLM 负担。
关键区别：向量检索是「Query、文档单独编码」Rerank 是「Query + 文档拼接交叉编码」，匹配精度碾压向量
2. 常见重排序方案

（1）规则重排（轻量）

按文档时间、长度、权重、关键词命中数、来源权重过滤
耗时极低：0～3 ms
适合低资源、简单业务
（2）交叉编码器 Rerank 模型【工业主流】

代表模型：BGE-Reranker、BCE-Rerank、M3E-Rerank、Jina Rerank、Cohere Rerank
输入格式：[CLS] 问题 [SEP] 文档片段 [SEP]
直接输出「相关性分数」，精准排序
RAG 提升最大的关键模块
（3）LLM 重排（小众）

让大模型逐条判断文档相关性、打分筛选
效果最好
代价极高、耗时巨长，一般线上不使用
3. 耗时范围（线上实测）

规则重排：0～3 ms
GPU 部署轻量 Rerank 模型：5～20 ms
CPU 部署 Rerank 模型：30～100 ms
LLM 重排：200～800 ms+
4. 核心特点

精度极高、强降噪、显著提升回答准确率
推理有算力消耗，不能海量数据全量跑
必须依赖「初检索缩容」才能落地
四、初检索 VS 重排序 核心对比

表格
维度	初检索（粗排）	重排序 Rerank（精排）
处理数据	百万级全知识库	仅 Top30~100 少量候选
优化目标	速度 + 召回率	精度 + 准确率
匹配能力	浅层词法 / 语义	深层交叉语义理解
算力消耗	极低	中等
延迟	低	偏高
输出结果	多而杂	少而精
必要性	必选	可选，高端 RAG 必加
五、工业标准组合 & 整体耗时

标准流程

Query 改写 / 扩展
混合初检索：BM25 + 向量 → 合并去重 Top-50
Rerank 重排 → 精选 Top-6
上下文压缩 + 截断
送入 LLM 生成
全链路总耗时

无重排极简 RAG：20～60 ms
标准企业 RAG（GPU + 混合召回 + Rerank）：50～120 ms
CPU 低配部署 RAG：150～300 ms
六、面试高频简答（背诵版）

为什么需要初检索？海量知识库无法全量高精度计算，通过轻量化算法快速缩小候选范围，保证召回率与响应速度。
为什么一定要用 Rerank？向量 / 粗排打分粗糙、噪声多；Rerank 通过交叉编码深度理解语义，过滤无效文档，大幅提升 RAG 答案质量。
为什么不能直接全量重排？Rerank 模型计算开销大、延迟高，全量检索会导致接口超时、并发崩溃，必须先粗排缩容。
混合召回优势？BM25 擅长关键词与实体，向量擅长语义泛化，两者融合兼顾召回与语义匹配。
